# Smart-Cat 脚本参数说明

本文档用于说明项目中常用脚本的运行方式和可选参数。README 只保留最常用命令，本文件作为“参数字典”，方便第一次接触项目的人查阅。

## 参数使用原则

优先使用一键入口：

```bash
./bin/run_smart_scale.sh
./bin/run_smart_scale_service.sh
./bin/run_voice_command.sh
./bin/run_voice_command_mqtt.sh
```

这些入口会自动切换到项目根目录，并使用 `.venv/bin/python` 运行对应 Python 脚本。大多数运行参数已经写在：

```text
config/device_runtime.json
config/device_policy.json
config/mqtt.json
config/products.json
```

只有调试、演示或临时覆盖参数时，才需要手动加命令行参数。

## 一键入口脚本

| 脚本 | 作用 | 实际调用 |
| --- | --- | --- |
| `bin/run_smart_scale.sh` | 单次完整交易 | 先拉取 MQTT retained 配置，再运行 `scripts/smart_scale_demo.py` |
| `bin/run_smart_scale_service.sh` | 常驻称重触发服务 | 运行 `scripts/smart_scale_service.py` |
| `bin/run_voice_command.sh` | 本地语音补盲命令 | 运行 `scripts/voice_accessibility.py` |
| `bin/run_voice_command_mqtt.sh` | MQTT 语音命令监听 | 运行 `scripts/voice_command_mqtt.py` |

一键入口后的参数会原样传给对应 Python 脚本，例如：

```bash
./bin/run_smart_scale_service.sh --heartbeat-interval 60 --verbose
```

## 常驻称重服务

脚本：

```bash
./bin/run_smart_scale_service.sh
```

对应 Python：

```bash
python scripts/smart_scale_service.py
```

常用参数：

| 参数 | 默认值 | 作用 | 何时调整 |
| --- | --- | --- | --- |
| `--weight-threshold-g` | `30.0` | 超过该重量认为有物体放上秤 | 放很轻商品时可调低 |
| `--empty-threshold-g` | `20.0` | 低于该重量认为秤台为空 | 空秤仍有残余重量时可调高 |
| `--stable-samples` | `5` | 连续多少次读数满足稳定条件才触发交易 | 重量抖动时可调大 |
| `--stable-delta-g` | `3.0` | 稳定窗口内允许的最大重量波动 | 秤体抖动时可调大 |
| `--poll-interval` | `0.3` | 轮询称重间隔，单位秒 | 想更快响应可调小 |
| `--cooldown` | `2.0` | 交易完成后进入等待拿走前的冷却时间 | 防止重复触发 |
| `--sync-interval` | `15.0` | 空秤时同步远程配置/策略的间隔 | 演示频繁改参数时可调小 |
| `--heartbeat-interval` | `60.0` | 设备心跳上报间隔，`0` 表示关闭 | 终端或 Web 状态刷新调试 |
| `--once` | 关闭 | 完成第一笔交易后退出 | 单次测试常驻逻辑 |
| `--verbose` | 关闭 | 显示 MQTT/RKNN/OpenCV 详细日志 | 排查底层问题时打开 |
| `--no-sync` | 关闭 | 不从 MQTT 拉取运行参数和策略 | 离线测试时使用 |
| `--subprocess-transaction` | 关闭 | 每笔交易用子进程运行单次脚本 | 回退隔离调试 |

不常改但存在的底层参数：

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `--runtime-config` | `config/device_runtime.json` | 板端运行参数文件 |
| `--device-policy` | `config/device_policy.json` | 设备策略文件 |
| `--mqtt-config` | `config/mqtt.json` | MQTT 配置文件 |
| `--hx711-config` | `config/hx711_scale.json` | HX711 校准参数 |
| `--gpio-backend` | `gpiod` | GPIO 访问方式 |
| `--samples` | `5` | 服务轮询时每次读取的 HX711 样本数 |
| `--sample-interval` | `0.02` | HX711 样本间隔 |
| `--max-deviation` | `5000.0` | HX711 原始值滤波偏差 |
| `--events` | `records/device_events.jsonl` | 设备事件本地记录 |
| `--status-file` | `records/service_status.json` | 当前服务状态快照 |

示例：

```bash
./bin/run_smart_scale_service.sh
./bin/run_smart_scale_service.sh --once
./bin/run_smart_scale_service.sh --heartbeat-interval 0
./bin/run_smart_scale_service.sh --weight-threshold-g 20 --stable-delta-g 5
./bin/run_smart_scale_service.sh --verbose
```

## 单次完整交易

脚本：

```bash
./bin/run_smart_scale.sh
```

对应 Python：

```bash
python scripts/smart_scale_demo.py
```

常用参数：

| 参数 | 默认值来源 | 作用 |
| --- | --- | --- |
| `--runtime-config` | `config/device_runtime.json` | 运行参数文件 |
| `--device-policy` | `config/device_policy.json` | 策略文件 |
| `--accept-confidence` | `0.75` | 自动结算所需 Top1 置信度 |
| `--confirm-gap` | `0.15` | Top1/Top2 差值低于该值则待确认 |
| `--tts-volume` | `3` | SYN6288 音量，范围 `0-16` |
| `--tts-speed` | `5` | SYN6288 语速，范围 `0-5` |
| `--mqtt` | 配置文件 | 是否上报 MQTT |
| `--mqtt-optional` | 配置文件 | MQTT 失败是否不阻断交易 |
| `--no-record` | 关闭 | 不写入交易 JSONL |
| `--json` | 关闭 | 输出 JSON 结果 |

摄像头参数：

| 参数 | 默认值 | 作用 |
| --- | --- | --- |
| `--device` | `0` | 摄像头编号或 `/dev/video0` |
| `--image-output` | `outputs/current.jpg` | 拍照输出路径 |
| `--width` | `1920` | 请求图像宽度 |
| `--height` | `1080` | 请求图像高度 |
| `--warmup` | `1.0` | 摄像头预热时间 |

识别参数：

| 参数 | 默认值 | 作用 |
| --- | --- | --- |
| `--model` | `models/yolo_product_detector.rknn` | RKNN 模型路径 |
| `--labels` | `models/yolo_product_detector.labels.json` | 标签文件 |
| `--products` | `config/products.json` | 商品表 |
| `--topk` | `3` | 记录前几个候选商品 |
| `--rknn-imgsz` | `640` | 模型输入尺寸 |
| `--rknn-layout` | `nhwc` | RKNN 输入布局 |
| `--rknn-float-input` | 关闭 | 是否使用 float32 输入 |
| `--det-conf` | `0.25` | YOLO 检测框过滤阈值 |
| `--det-iou` | `0.45` | NMS IoU 阈值 |
| `--det-max` | `20` | 最大保留检测框数量 |

称重参数：

| 参数 | 默认值 | 作用 |
| --- | --- | --- |
| `--hx711-config` | `config/hx711_scale.json` | HX711 校准参数 |
| `--gpio-backend` | `gpiod` | GPIO 后端 |
| `--samples` | `20` | 单次交易称重采样数 |
| `--max-deviation` | `5000.0` | 原始值滤波偏差 |
| `--weight-g` | 无 | 直接指定重量，跳过 HX711 |
| `--weight-raw` | 无 | 与 `--weight-g` 配套记录原始值 |
| `--weight-source` | `hx711` | 重量来源标签 |

语音参数：

| 参数 | 默认值 | 作用 |
| --- | --- | --- |
| `--tts` | `syn6288` | `none` / `mock` / `syn6288` |
| `--tts-port` | `/dev/ttyS10` | SYN6288 串口 |
| `--tts-baudrate` | `9600` | 串口波特率 |
| `--tts-encoding` | `gb2312` | 文本编码 |
| `--tts-music` | `0` | 背景音乐编号 |
| `--tts-music-volume` | `0` | 背景音乐音量 |

示例：

```bash
./bin/run_smart_scale.sh
./bin/run_smart_scale.sh --accept-confidence 0.75 --tts-volume 2
./bin/run_smart_scale.sh --weight-g 500 --weight-source manual_test
./bin/run_smart_scale.sh --json --no-record
```

## HX711 称重测试与校准

脚本：

```bash
python scripts/hx711_reader.py
```

常用参数：

| 参数 | 默认值 | 作用 |
| --- | --- | --- |
| `--config` | `config/hx711_scale.json` | 校准文件 |
| `--gpio-backend` | `gpiod` | GPIO 后端 |
| `--samples` | `10` | 读取样本数 |
| `--interval` | `0.05` | 样本间隔 |
| `--max-deviation` | `5000.0` | 滤波偏差 |
| `--print-samples` | 关闭 | 打印每个原始样本 |
| `--tare` | 关闭 | 当前空秤作为零点 |
| `--known-weight-g` | 无 | 已知砝码重量，用于计算比例 |
| `--min-calibration-delta` | `100.0` | 标定时要求原始值至少变化多少，防止砝码未放好就保存错误比例 |
| `--save-config` | 关闭 | 保存校准结果 |
| `--raw` | 关闭 | 只输出原始均值 |

引脚参数：

| 参数 | 默认值 | 作用 |
| --- | --- | --- |
| `--dout-gpio` | `134` | sysfs DOUT GPIO |
| `--sck-gpio` | `132` | sysfs SCK GPIO |
| `--dout-chip` | `gpiochip4` | gpiod DOUT chip |
| `--dout-line` | `6` | gpiod DOUT line |
| `--sck-chip` | `gpiochip4` | gpiod SCK chip |
| `--sck-line` | `4` | gpiod SCK line |

示例：

```bash
sudo .venv/bin/python scripts/hx711_reader.py --tare --save-config
sudo .venv/bin/python scripts/hx711_reader.py --known-weight-g 500 --save-config
sudo .venv/bin/python scripts/hx711_reader.py --samples 20 --print-samples
```

